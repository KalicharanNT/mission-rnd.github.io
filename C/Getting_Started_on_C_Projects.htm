Getting Started on C Projects

Overview

The following steps will walk you through the setup for C projects:


Pre Requisites

  1. Install git from http://git-scm.com/downloads  and allow the installer to add git.exe to your PATH environment variable.  

    To configure the paths manually, add <InstallationDir>/Git/bin and <InstallationDir>/Git/usr/bin to the PATH environment variable.

    Configure git with your username and email with following commands.

    git config --global user.email "<EMAIL>"

    git config --global user.name "<USERNAME>"

    TIP: After installation is successful open command prompt and type "git". You should see git usage commands.

    TIP: Also check ssh in the path. For that, open command prompt and type "ssh". You should see ssh usage commands.

    Follow this link to add environmental variables for enabling ssh.

  2. Signup for a github account at: http://github.com/ if you dont have a github account
  3. Generate SSH keys While generating SSH keys, keep the pass phrase empty
  4. Install Visual Studio Express 2013 for Windows Desktop.

Steps to run tests in visual studio

  1. Open visual studio after installation is successful.
  2. Select FILE --> Open Project. Choose .sln file of the project.
  3. You will see spec and src projects. spec project contains all test files and src project contains all source files.
  4. Now run tests from TEST --> Run --> All Tests. You will see test results on left pane.
  5. If you want to debug any test, keep a breakpoint in the file you want to debug and run test from TEST --> Debug --> All Tests.

Getting Started with the tool

  1. For the initial launch, github username and password have to be provided to the tool configuration settings.

  2. Following launch screen will appear once the tool is configured with required github credentials.

  3. Click on create new project and select the project that you want to start working with

  4. The project home screen shows the activities that are completed and the activities that haven't been completed. It has the options to test and submit your code.

  5. Open the cloned project in Microsoft visual studio 2013 and start completing activities in the required order.
  6. Complete the assignments by coding the activities in the project directory. Each project is broken into a set of activities. Once you have coded an activity, you can verify if your code is correct by running predefined test cases. The pre-defined tests are under the spec folder.
